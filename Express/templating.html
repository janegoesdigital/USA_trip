What is templating

Templatng means that you don't have to worry about switching languages.

Can write in your target language AND have the ability to insert dynamic data without having to go out to JS. Recommends avoiding writing HTML in JS

    types of template engines include
  • mustache
  • jade
  • handlebars

    • Can render your template option

    • client side; or
    • server side ie render HTML beforeit is sent to the client. Hides details and supports caching - important for performance

    How does it work?

    Need to understand context. When you render a template, you pass the templating engine an object called the context object and this is what allows the replacement to work

    Views and layouts

    What are they

    view usually represents individual page on website.

    Layout - special kind of view - a template for templates

    View is rendered BEFORE the layout. Enables view to customise the layout, which becomes important for sections

    Making sure they are used - file structure and app.get code

    Views go in the views main directory. Layouts, go in a layout subdirectory

    can render a view from foo.handlebars file this way:
    app.get('/foo', function(req, res{ res.render('foo'); })); Will default to the main layout. Can also specify layout none or a different layout

    Partials

    ie reusable compontents - known as widgets in front end circles

    To get a partial up and running need to:

    1. create the partial in the partials file
    2. add middleware to the app file to get the partial
    3. put the partial on a page
    To include a partial on a page use {{> partial_name}}

    Sections

    Sections deal with the scenario where your view needs to inject itself into different parts of your layout eg add something to the head element or insert a script that uses jQueyr

    No built in way to do this in handlebars or exrpess - but can use Handlebar helpers - adding an object we'll call section. See the notes
